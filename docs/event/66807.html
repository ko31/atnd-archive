プログラマ、エンジニアの皆さん。
コードレビュー 実践できていますか？

  • 「チーム開発しているけど、他人のコードは読まないなー」
  • 「コーディング規約は用意けど、誰も守ってくれない」
  • 「チームのスキルレベルがばらばらで、なかなか底上げしない...」

という問題は、コードレビューを導入することで改善できます。

コードレビューを、無理なく開発に取り入れるための 最良の方法は
GitHub の Pull Request（プルリクエスト）を使うことだと思います。

　

その Pull Request を体験してみませんか？( ･`ω･´)

　

どんなことをやるの？

発表会 + レクチャー（ハンズオン）

前半：Pull Request を使ったコードレビューのやり方について説明します。
後半：各自 PC で実際に Pull Request を試してみます。

私の方で、
簡単な HTML と JavaScript のコードを準備します。

それを実際に変更、git で操作しながら、
Pull Request の作成 → コードレビュー
といった流れを体験することができます。

  • git
  • GitHub（GitHub Flow）
  • Pull Request

↑ この辺りに、興味のある方は是非！

　

参加対象者

  • ソフトウェアプログラマ、エンジニア
  • Pull Request を使ったことがない、GitHub Flow を知らない方

コードレビューのメリットを感じていただくことを目的としますので、
とても、とても簡単な所からスタートします。

　

初心者の方、
人にコードを読まれるのは恥ずかしい (._.) というシャイな方、
いままで勉強会に参加したことがないという方でも、
お気軽にお越しください。

　

見学のみの参加も大丈夫です。
（※見学希望とコメントしてください）
