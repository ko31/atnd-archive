[スケジュール]
18:30～ 開場
18:40～19:00 … ヨーロッパお勧め観光情報
6時半までに職場を抜け出して会場に着くのは厳しい！そんな人の
ために、最初は少し気楽なテーマでお話しします。
早く到着された方は、一度と言わず、二度三度と訪れておくべき
素敵な（が、少しマイナーな）場所をご紹介します。
19:00～20:00 … PG-Strom – GPUがPostgreSQLを加速する
PG-Stromとは、PostgreSQLのSQL/MEDを利用した拡張モジュールで、
巨大なテーブルに対するDB検索負荷の一部をGPGPUにオフロードし、
特に複雑な条件句を含む検索処理を高速に処理する事ができます。
本発表では、GPUの特性やプログラミングスタイルを紹介すると共に、
DB検索ワークロードの特徴からCPU/GPUがどのように役割分担を行う
べきか、そしてPG-Stromがどのように負荷をGPUにオフロードしているの
かを説明します。
また、列指向データ構造、非同期処理、GPUコードの動的生成とJITコン
パイルといったPG-Stromの軸となるアイデアを説明し、今後の方向性に
ついてもご紹介します。
20:00～20:20 … 行レベルアクセス制御 – v9.3での実装と技術的背景
PostgreSQL開発者コミュニティでは、v9.3の開発に向けて大詰めの議論が
続けられています。大物機能の一つが、行レベルアクセス制御機能です。
行レベルアクセス制御とはどういったものか？そして、どういった課題が
あり、コミュニティはこれらの問題をどのように解決していったのかを
簡単にご紹介します。
20:20～21:30 … 懇親会
会場でのビアバッシュを予定しています。
講師の手荷物に余裕があれば、ドイツから何か持ち込むかもしれません。
[講師紹介]
海外 浩平（KaiGai Kohei / kaigai@kaigai.gr.jp / Tw: @kkaigai)

SELinuxを軸にLinux kernelやPostgreSQLなど諸々のコミュニティで地味に
活動しているOSS開発者。v9.1で標準機能化されたSE-PostgreSQLの開発や、
Webスタックのアクセス制御を一元化するLAPP/SELinuxの開発が主な仕事。
2011年よりNEC Europeに出向。ドイツでSAP-AGとのアライアンスを手掛ける
一方、GPUを利用してDB検索を高速化するPG-Stromを新たに開発。
今回の勉強会は、PG-Strom本邦初公開となります。
