バックエンド トリセツ #1 : Bull Session

サーバサイドエンジニアをメインジョブに活躍しているみなさまへ。

最近、オーダーと守備範囲があっていないことがありませんか？
マークアップをやったり、インフラもお願いされたり…

クリエイターにとって「やってやれないことはない」という意気込みは忘れたくないですが、
そういう問題じゃないし！
みなさまがどう解決されているかを共有できるだけでも、明日への自信に繋がります。

「オーダーミス」を未然に防ぐ、サーバーサイドエンジニアとりせつ、
まずはこのような具体的なテーマの元、わいわいできればと思っています。（ mackarel Inc. 服部生実 ）

▼ アジェンダ

  • ReactなどモダンなSPAライブラリ、フレームワークについて
    • ReactってOSSの団体から村八分だとからしいけどどうなの？
  • nginxの設定の書き方
    • 特にリダイレクト設定、特に特にプロキシ経由のリダイレクト
  • フロントエンドとは違うんですけど、、な依頼が来たことありますか？
    • ある場合、どうやって切り抜けましたか？
  • というかそもそもどういう話がしたいです？
    • 次回があればw、次はそういうのから拾ったり

▽ Bull Session について

毎日への退屈を「たまに」どうにかしたくなりませんか？
セミナー等の参加モチベーションに「ムラ」がありませんか？
でも何とかしたい時がある。

そういう時だけ集まれて、次の日からちょっと元気になれる集まりができればと思い、
ざっくばらんに雑談という意味の Bull Session をやっています。

ニーズがないとやりませんが、少しでも声が上がればいつでも開催します。
議論の加速が止まるのも嫌だから SNS で大きく拡散することも（基本）やりません。
でも、いちげんさん大歓迎です。

ライトな気持ちで、たまには脱線しまくったりして、
でもその場だけではない、次に繋がる会を続けれるよう、モデレーター頑張りますので、
お気軽に参加してもらえると嬉しいです。
