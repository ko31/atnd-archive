GitHub ハンズオン

TDDBC in Fukuoka でペアプロを行うのですが、github使ってコードを共有すれば、(特にrubyとかLLを使ってる場合)宗教戦争を回避できると信じてこのイベントをやります。

もちろんTDDBC参加者でなくても参加OKです！

今回は、githubのみ扱います！！(ばらばらにすると色々面倒なので)

ハンズオンの内容は、二人一組のチームでgithubを使いコードを共有する体験してもらいます。(ざっくりと)

ハンズオンの内容

バージョン管理やgitの説明

ここはSpring_MTがします

一人でgitの環境を触る

  • ローカル環境でgitのリポジトリを作ってgitの環境に慣れる
  • ローカルの環境をgithubと同期する

ペアでgitの環境を触る

  • pull requestを使った同期
  • hubコマンドを使ったissueとの紐付け

対象ユーザー

  • コマンドライン(黒い画面)をある程度使える人(今回はgithubのGUIアプリを使う予定はないです)
  • git/github初心者

準備してきて欲しいもの

  • gitのインストール
  • githubのアカウント

マシンはgitが使えればなんでもOKです
