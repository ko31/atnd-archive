TDD Boot Camp 札幌 2.1

TDD Boot CampはTDD（テスト駆動開発）を実際に体験しながら学ぶイベントです。
これまで東京を初め北陸や名古屋でも開催されており、札幌では通算で４回目の開催となっています。
今回はこれまでのTDDBCと少しだけ趣向を変え、「開発現場でTDDを実戦する」をテーマに
比較的に身近であると思われる業務系ウェブアプリケーションをTDDを使って体験するイベントにしました。

参加者は言語/フレームワーク毎に４名程度のチームに分かれ、「会議予約システム」の開発を行います。
システムの大まかなユースケースやモックアップは運営側で用意しています。
各チームはそのユースケースから必要な振る舞いを確認し、小さなステップで１つづつテストを書きながら開発をします。
実装するシステムの難易度としては簡単なものですが、TDDを適用してみることでこれからの開発への大きなヒントとなるでしょう。

対象者

  • テスト駆動開発に興味のあるプログラマ
  • 業務システムに携わる開発者
  • ビヘイビア駆動開発やユースケース駆動開発に興味のある開発者

尚、プログラミング経験としては最低限のデータ構造・データベース・言語毎の構文・ウェブアプリケーションの仕組みを理解している事が望ましいですが、TA（ティーチングアシスタント）が各チームに１名ずつ配置されるため、高い経験や高度な知識は不要です。

演習に関して

  • ３〜４名で１チームで行います。
  • １台のPCを使って、TDDの１サイクル毎に実装者（ドライバー）を交代します（乱取り形式）。
  • PCは各チームに１台あれば良いので、全員が持ってくる必要はありません
  • 友人同士などの参加の場合、必ずしも同じチームに配置されない可能性があります
    • 友人同士２名＋他の人２名はOK、友人同士３名＋他の人１名は不可など
  • 各言語毎にサポートスタッフを１名以上配置する予定です
