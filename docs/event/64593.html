今回の趣旨

前回、大好評だった「異なる開発方法論の実務者によるモデリング発表会」の第二弾です。前回と同じ「 花束問題V1.2 」を違う手法で発表して頂きます。今回もちょっと贅沢すぎるような面々にお越しいただけます。前回も満員でお断りした方もいらっしゃいましたのでぜひ早い目の参加表明をお願いします。

★製本できるように編集しました。１枚目が表紙＋裏表紙になります PDF版 DOC版

ドメイン駆動設計

　　　インクス株式会社 社長　後藤 秀宣さん

WEB+DB PRESS誌に「巨人の肩からPHP」を連載されていたのでご存じの方も多いと思います。大阪の久保敦啓さんとともに「 PHPメンターズ 」を立ち上げ、ドメイン駆動設計のトレーニングもされています。ドメイン駆動設計をこのIT勉強宴会で取り上げる時に久保さんと共に講師をしてもらい知り合いました。その後何度もIT勉強宴会にお越しいただいています。
私自身、実際の業務に即したDDDモデルは見たことがありません。有償でセミナーをされている内容のエッセンスを実際のモデルを元に解説してもらえそうです。どういうモデルになるのか楽しみです。

ドメインクラス図

　　　有限会社井田　　 社長　井田　明男さん

「かんたんUML」で有名な（株）オージス総研に23年おられて独立されました。そのころに渡辺幸三さんに紹介されて知り合いました。今は同志社大学の講師もされています。
UMLのモデリングについては米国から翻訳された様々な手順が広がっています。ところが、英語と日本語の言語としての違いが大きく、そのまま日本語に訳しても実は理解出来ないのではないかという問題意識を持たれています。つまりこの花束問題のように，自然な日本語で論述された要求はクラス図化しにくいのです。ではどうすれば良いのかという点について同志社大学の学生さんに講義されているエッセンスをお話しいただけます。初学者の方には特に勉強になるのではないでしょうか。楽しみです。

データモデル　THデータモデル

　　 株式会社データ総研 　　社長　堀越　雅朗さん
　　 株式会社イントリーグ 　石井　健作さん

DOAという言葉をご存じでしょうか？データ中心アプローチ（Data Oriented Approach）という和製英語です。そのDOAの元であるTH法を作られた椿正明さんがその手法を使って企業の役に立つために作られた会社がデータ総研です。今年はなんと30周年を迎えられます。もし日本でTHデータモデルが生まれなければ、今のような日本企業の発展がなかったかも知れないとまで思っています。
モデリング競演には是非TH法もお願いしたいと思い社長に直談判しました。会社では有償セミナーをされている内容ですが自ら解説していただけることになりました。石井さんは元データ総研で堀越さんの部下だった頃からの知り合いです。今ではデータモデリングの知識を生かしてRFPコンサルタントとして活躍されています。実際のモデルはその豊富な経験を活かして石井さんが作成してくださるとのことでした。
前回ご登壇いただいた渡辺幸三さんも鳥谷部聡さんもＴＨ法を良く勉強されています。基礎教養としてもTHデータモデルを知り勉強して欲しいと思います。
