コンテンツ

WordPressの開発環境にどんなツール使ってますか？
XAMPPやMAMPですか？
XAMPPやMAMPは確かにインストールも簡単だしすぐに開発環境作れますね。
でも、次のようなトラブルないですか？

・本番と動作環境が違う
・チーム内で動作環境が違う
・開発環境の構築が複雑化してきた
・SSL環境のチェックができない

などなど
ということで上記を解決できる「Vagrant」での開発環境構築をハンズオン（体験しながら学ぶ）します。

ハンズオンですのでご自身のPCを持参してください。
サポートしますので実際に環境を構築しながら覚えていけます。
スライドも作って説明もしますのでPCは必須ではありません。
「Vagrant」って何？って聞きたいだけの人も大歓迎！

Vagrant 構成は「VCCW」を使います。
http://vccw.cc/

【おまけ】
時間があれば以下の内容をセッションします。

以下のようなこと経験ありませんか？

・本番からローカル環境へ丸ごととサイトをコピー
・ローカル環境で作ったサイトを本番へ
・サーバ移転
などなど

Codexの「サイト URL の変更」http://goo.gl/H9sbu
でも手順が記載されているとおり結構面倒ですよね。
でもページの一番下に「WP-CLI」の欄があります。
数行で終わってますね。
「WP-CLI」を使えば非常に簡単にURL変更できます！
「Vagrant:VCCW」で構築するとWP-CLIというツールが使えるようになるのでこの方法が開発環境で使えるようになります。
