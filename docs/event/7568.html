ちょっと成長してきた？MessagePackを、「コレは使いたい！」へとレベルアップさせるべく…

内容

  • MessagePack ＆ MessagePack-RPC を他の言語に移植する
    • C#とかScalaとかJRubyとか…
  • RPCとIDLの仕様を作り込む
    • エラーの返し方（シリアライズの方法・エラーの継承関係の表し方）
    • 例外クラスを生成するIDLの仕様
  • テストを書く：異常系のテストはMUSTだと聞きました…が！無いorz
  • MessagePackを使ったシステムやソフトウェアを作る
  • MessagePackを既存のシステム（や言語）に導入する
  • ドキュメント書く：足らない
  • 宣伝する
  • ウチのココで使いたい/使っているなど。情報交換

などなど。

MessagePackのリポジトリはこちら：http://github.com/msgpack
