概要

  • チームkatagaitaiの実施する，CTFに関する勉強会です．過去問を解きつつ技術解説をします．
  • 学生～社会人若手の方を対象にわいわいやりたいと思ってます．
  • 未定箇所については決まり次第ATNDを更新しますので，こまめにチェック頂くのが良いかと思います．

ターゲット層

  • Crypto, Pwnableに興味のある， 中級者くらいの人 や 強くなりたい人 を対象にしています．
    • 初心者向けではありません が，聞くだけ聞きたい～という方であれば初心者でも参加はWelcomeです．
    • 中級者の定義が曖昧ですが，その辺は今回の問題や過去資料を見て察して下さい．
    • 当日の雰囲気については 去年のまとめ が参考になると思います．

参加資格

  • 以下の1～3いずれかに当てはまる方
    1. 学生の方
    2. 若手の方
    3. 気持ちが若手な方

持ち物

  • ノートPC
  • ネット回線(テザリング等)

費用

  • 無料
    • 学生の方も社会人の方も，出来ればWikipedia等お好きなWebサービスやOSSに1000円寄付をお願いします．
    • (勉強会が終わったあとでも良いので)

服装

内容

crypto

  • hack you 2014 – Crypto200 – hashme
    • hash length extention
  • ebCTF 2013 – Bin400 – md5colliding
    • chosen prefix collision
  • おまけ
    • Plaid CTF 2014 – Crypto250 – parlor

pwnable

  • PlaidCTF 2015 – Pwnable620(420+200) – tp
    • リピーターの方も多く，皆さんもpwnに慣れてきたと思うので，前回より難し目のヒープ+sandbox問を取り上げています．
    • heap
      • thread arena
    • sandbox
      • seccomp
    • thread
      • clone
    • おまけ(多分説明はカット)
      • ucontext
      • DWARF (触りだけ)

準備しておくと良いもの

crypto

  • x86のPEバイナリが動くWindows環境
  • x86/x64のELFバイナリが動くLinux環境
    • Ubuntu14.04 x64 想定です．boostが利用できるよう，以下のセットアップが必要です．
      apt-get install libboost-all-dev lib32z1-dev libbz2-dev
  • python(2系)の知識
  • 簡単な暗号の知識
    • 直接関係はしませんが， 過去資料 の#4には目を通しておくと良いかもしれません．

pwnable

  • Ubuntu14.04.4 x86_64のLinux環境
    • ただし kernel3.19.0-41以降の環境では問題バイナリが正常動作しません．
      • 原因は PIEの動作変更によるもの で，ubuntuでは3.19.0-41以降のカーネルに導入されています．
      • @Ga_ryo_さん，@superbackerさん，問題報告/対処方法の連絡ありがとうございました．
    • 各種ツールのセットアップは こちら です(追記して長くなったのでpastebinに移しました．7/2更新)
      • @K_atcさんが vagrantで環境構築 できるよう作ってくれました．こちらを利用しても良いでしょう．
  • x86_64アセンブリの知識
    • 当日バイナリを読む時間は確保していません．各自しっかり読んでおくと良いでしょう．
    • 解析のヒント
      • ヒント1 ヒント2 ヒント3 ヒント4
    • 過去の勉強会で解説したことは基本的に端折ります．
      • 特にヒープのチャンク構造，Use After Free，DynamicROP辺りはある程度知っている前提です．
      • 必要に応じて 過去資料 の#1，#2，UAF勉強会 に目を通しておきましょう．
  • python(2系)の知識
