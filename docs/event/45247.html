技術の高さだけでは、クライアントは満足しない

マークアップに要求されるのはクオリティとスピードですが、実務ではそれが全てではありません。
クオリティ（コードの正確性）は突き詰めれば同じようなコードに収束します。
スピードはスケジュール通りに終えることができれば問題ありません。

ではクライアントには何をアピールすれば良いか？

本セミナーでは、マークアップにおけるテクニックやノウハウではなく、
「どのような思考でマークアップを行うか」という内容をメインにお話いたします。

ハンズオンでは、座学を踏まえた実際の制作工程を実践します。
仕様書・デザインを見ながら、実際にコーディングしてみましょう。
これらを実践することで、実務に直結する内容になるでしょう。

Web Developer Selection（WDS）とは

Web Developer Selection（WDS）は、「数多くある技術から、Web Developerが最適な知識や技術をセレクション。セッション+ワークショップで提供する、とことん実践主義にこだわった技術力向上を目指す場」です。
「レスポンシブWebデザインの実践ワークフロー」でDesign Spiceの小川さんや、「FireworksユーザーのためのPhotoshop講座」をStocker.jpのなつきさんに、また、「CSSレイアウトの理解と実践」ではKOJIKA17のこじかさんなど、多数の講師の方にお話頂いております。

講師プロフィール

朝倉裕也（そぷ）氏
2007年より約4年間、ゴメス・コンサルティング株式会社にてマークアップエンジニア・マークアップチームチーフとして従事。
その後、約1年間モーニングスター株式会社にてマークアップチームチーフとして従事後、独立。
現在はDIGITAL POPEYE（デジタルポパイ）のフリーランスマークアップエンジニアとして前線で活躍している。

HTMLやJavaScriptなど、フロントエンドを中心にした制作者で、Web標準に準拠した仕上がりのクオリティと、満足度の高い仕事振りに、顧客は高い信頼を寄せる。
RSSを検索・登録できる"わたしだけのアンテナサイト" – iSoople の制作・運営でも知られる。
