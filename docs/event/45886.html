ドメイン駆動設計を知ろう

ドメイン駆動設計（DDD: Domain-Driven Design）をご存じでしょうか？ Eric Evansが2003年に出した同名の著書で広まったソフトウェアの設計手法です。日本では2011年に 出版 され広がってきましたがまだまだこれからの技術です。この、ドメイン駆動設計のセミナーを開催されている二人が無料で基本を教えて下さいます。

この本を読んだ方も読んでいない方も、多数お集まり下さい。
もっと詳しく知りたい方は、PHPメンターズの有償トレーニングをご利用下さい。
http://phpmentors.jp/training

ドメイン駆動設計って何？

「ドメイン」とは「アプリケーションが対象とする業務領域」の事です。業務アプリケーションを設計する時、「プレゼンテーション層」「ドメイン層」「データソース層」と分けることは普通にされると思います。その根本になるのがドメインです。この分類は渡辺幸三氏の 三要素分析法 と通じるものがあります。

　プレゼンテーション層：機能構成
　ドメイン層：　　　　　業務構成
　データソース層：　　　データ構成

ＷＥＢアプリケーションを作成するときの「ＭＶＣ」という実装に近い分類と較べると相当本質をついているのだと思います。
