TDD Boot Camp 札幌 2.3

TDD Boot CampはTDD（テスト駆動開発）を実際に体験しながら学ぶイベントです。 これまで6回開催してきたTDD Boot Camp 札幌ですが、7回目の開催をいたします。

今回のテーマは「アジャイル開発におけるテスト」です。札幌2.1の時に好評だった、１つ上のレイヤーからテスト駆動開発にアプローチする、すなわち振る舞いや受入テストを意識したテスト駆動開発を実践します。今回も3〜5名程度のチームに分かれ、１つのプログラムを作成していきます。参加者の中には今回が初めてのテスト駆動開発かもしれませんが、少しずつ教わりながら体験してみてください。
また、2.2と同様に基調講演等には時間を多く割きません。初参加の人は、 和田卓人の“テスト駆動開発”講座 を観てから参加すると良いでしょう。

対象者

  • テスト駆動開発のスキルを高めたいプログラマ
  • テスト駆動開発に興味のあるプログラマ
  • アジャイル開発に興味のあるプログラマ・テスター
  • 自動化に興味のあるテスター

よくある質問

Q1. TDDBC初参加ですが大丈夫ですか？
A1. 大丈夫です。
ただし、当日はTDDに関する基調講演はありませんので、 和田卓人の“テスト駆動開発”講座 等をご覧になってから参加される事を強く推奨します。

Q2. 持ち運べる開発環境がありません
A2. 問題ありません。
演習はペアで行いますので、開発環境を持った人と組むことができます。ただし、使いたい言語やエディタについて自由に選択出来ない可能性がありますが、ご了承ください。

Q3. 予習は必要ですか？
A3. 予習することをオススメします。
演習用のお題について、各TDDBCにて実装コードが公開されてもいます。当日に右も左もよく解らない事が怖いのであれば、予習されると自信が持てるでしょう。テスト駆動開発とプログラミングに一定の自信があれば予習は不要です。
