TDD Boot Camp 札幌 2.2

TDD Boot CampはTDD（テスト駆動開発）を実際に体験しながら学ぶイベントです。 これまで5回開催してきたTDD Boot Camp 札幌ですが、6回目の開催をいたします。

今回のTDDBCのテーマは「基本に戻る」です。

TDDBC札幌ではから色々なお題を扱ってきました。1回目はWikiフォーマットのテキストをHTMLに変換するプログラム、2回目は各地のTDDBCでも行われているLRUキャッシュ、3回目は掲示板風プログラム、4回目はファイルIOを含むプログラム、5回目はウェブアプリケーションです。参加者もTDDに慣れてきた人も多いでしょう。複雑で難易度を高めたお題を行うのもありですが、慣れてきたからこそ基礎を振り返る事をテーマとします。また、不慣れな人に対しては積極的に自分の持つTDDのスキルを伝搬して欲しいというのも主催としての願いです。
また、TDDBC札幌2.2では、基調講演等には時間を多く割きません。初参加の人は、 和田卓人の“テスト駆動開発”講座 を観てから参加すると良いでしょう。演習ではペアでどちらもTDD初参加にはならないように考慮しますのでご安心ください。

対象者

  • テスト駆動開発のスキルを高めたいプログラマ
  • テスト駆動開発に興味のあるプログラマ

尚、今回はウェブアプリケーションなどよりもプログラム的にやや複雑なお題を使い、専属のTAも不在ですので、普段使い慣れた言語での参加を推奨します。開発環境などを自分のノートPCにセットアップしての参加が望ましいですが、難しい場合はアンケートにてお知らせください。

お題

お題として「 LRUキャッシュ 」を扱い、ペアプログラミングで行います。ただし、言語やテスティングフレームワークには制限はありませんが、ペアが組めない場合などは変更して貰う可能性があります。

よくある質問

Q1. TDDBC初参加ですが大丈夫ですか？
A1. 大丈夫です。
ただし、当日はTDDに関する基調講演はありませんので、 和田卓人の“テスト駆動開発”講座 等をご覧になってから参加される事を強く推奨します。

Q2. 持ち運べる開発環境がありません
A2. 問題ありません。
演習はペアで行いますので、開発環境を持った人と組むことができます。ただし、使いたい言語やエディタについて自由に選択出来ない可能性がありますが、ご了承ください。

Q3. 予習は必要ですか？
A3. 予習することをオススメします。
演習用のお題について、各TDDBCにて実装コードが公開されてもいます。当日に右も左もよく解らない事が怖いのであれば、予習されると自信が持てるでしょう。テスト駆動開発とプログラミングに一定の自信があれば予習は不要です。
