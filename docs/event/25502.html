グローバルブレインズ 「WaRKS」 紹介

  • “WaRKS”: とは
    WaRKS(Waiting Room Kiosk and Screen)とは
    病院や銀行の窓口などで使える受付管理システムのOSSです。
    従来の受付管理システムは「Push通知」や「音声呼び出し」など
    複雑な処理が必要な為に、VB.NetやJavaなどで構築されたものが多く、
    設置の費用や手間が掛かっていました。
    しかし、WaRKSはRails3.1の新機能を活用することで、
    簡単に高機能のシステムをブラウザベースで実現しています。
    【WaRKS デモサイト】
    受付画面（患者側）⇒ こちら
    管理画面（病院側）⇒ こちら
    管理画面から「アナウンス」「呼出」の操作を行うと
    push通知にて、受付画面の該当箇所が点滅し、
    アナウンスが流れます。
    専用ブラウザはChromeです。
    ※他のブラウザでは正常に動作しない場合があります。
    【WaRKS DLサイト(github)】⇒ こちら

ライトハウス「 eNote for Community 」紹介

  • “eNote”: とは
    eNote for Communityとは企業向けグループウェア＆営業日報システムです。
    Ruby on Railsをベースに開発されており、2010年2月にオープンソースとして公開しました。
    このグループウェアを「 eNote 」として、お気軽＆すぐに使えるように有料サービス（SaaS）も行っています。

公開依頼、累計で3400回以上のダウンロードが行われていますが、ライトハウスが取っているオープンソース戦略、そして悩みや問題点などをぶっちゃけます。

【ソース公開サイト(github)】⇒ こちら

ノーチラステクノロジーズ「 Tengine framework 」紹介

動的に変化するデータセンター (Programmable Data Center) を実現するためのソフトウェア「Tengine framework」(テンジン フレームワーク)をオープンソースとして公開しました。

  • “Tengine framework”: とは
    Tengine framework とは、分散環境を意識したイベントハンドリング機能を基盤にリソース管理、ジョブの制御を行うエンジンです。
    これらの機能を組み合わせる事で “動的に変化するデータセンター (Programmable Data Center)” を実現することを目指しています。
    Tengine framework は、全てRubyで記述されています。
  • ドキュメント
    http://tengine.github.com/
  • ソースコード
    https://github.com/tengine
  • Tengineを使ったサンプルアプリケーション
    tengine_example
    https://github.com/tengine/tengine_example
