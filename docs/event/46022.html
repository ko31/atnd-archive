tech.guitarrapc.com – 第一回 PowerShell 勉強会を開催時のセッション資料公開 と舞台裏とか

開催概要

2013/12/21 (土) 六本木で PowerShell勉強会をやります。
少しでも興味があれば誰でも参加OK、PowerShellに焦点をあてた希少なIT勉強会です。

これが、 Japan PowerShell User Group の初セッションです。これからも、PowerShellを一緒に楽しむ機会を作れればと思いますです！

  • 定員を30名から40名に増員しました！
  • 前回、需要があるかアンケートを取った様子は PowerShell勉強会 仮 を見てください
  • 当日のUST中継などは今回は見送りとなります。(機材もってないです)

勉強会当日について(12/21 AM2:40 追記)

受付の場所
第一回PowerShell勉強会は、12/21 PM0:30 – 六本木ヒルズ森タワー LLフロアにて受付が立っております。HNにて本人確認させていただき、入館シールをお渡しします。案内のものが開場フロアまでご案内しますので、警備員のいる入場口から入ってください。

森タワーはどこ
六本木ヒルズ森タワーは、 http://www.roppongihills.com/facilities/access/train.html でいう日比谷線からのぼりエスカレーターで直通ルートがもっともわかりやすいかと思います。
エスカレーターを上って、まっすぐ目の前にある正面口が、オフィスフロアへの入口でオフィス入口のセキュリティゲート前に勉強会受付が立っています。

万が一、遅れてしまう場合
12:30～入場
13:00～ 開始
上記と13:00～30分程度(一回目のセッションぐらい) は、LLフロアに受付が待機しております。
その後は、Twitter で #JPPOSH のタグを付けて、@guitarrapc_tech までメンションをお願いします。スタッフのものがLLに迎えにまいりますので、恐れ入りますが少しお待ちいただきますようお願いします。

なるべく不自由がないように、行う所存ですがビル側のセキュリティと協力してスムーズな進行を心がけます。

「所属先と氏名」確認へご協力いただく必要があります。

ビル様のセキュリティの都合上、「所属先（個人可）と氏名」をいただく必要があります。
いただいた情報はビルに提出することのみに利用し、他の目的で利用することはありません。

恐れ入りますが、参加者の方に管理者からメールを送付いたしますので、返信にて「所属先（個人可）と氏名」のご提出をお願いいたします。
遅くとも、前日までの返信が必須となりますのでよろしくお願いします。
大変心苦しいのですが、どうかご協力のほどお願い申し上げます。

PowerShellの概要と使いどころ

登壇者 詳細
ぎたぱそ
(@guitarrapc_tech)		 PowerShellの歴史と目的
基本的な文法(ifやforeachなど)
cmdやvbs、シェルスクリプトとの違いから見る使いどころ

無題

登壇者 詳細
@SQLAzureJP なんとなくPowerShellコマンドの断片を知っていたり、
触ったりしたことはあるけど、ちゃんと使ったことが無かったのです。
IISの生ログを解析してねっといわれたら、
最近だとCentOSからアクセスできる場所において、awkでやっちゃう系。
そんな初心者がうっかり登壇すると言ってしまって、
泣きながらお勉強した10時間をお届け。
歴戦の兵士が居る前に喋るなんて、まぞっけたっぷりですが、
根気強くお付き合いください。

AWS Tools for Windows PowerShellをつかいこなそう

登壇者 詳細
渡邉源太(@gentaw0) AWS Tools for Windows PowerShellをつかったAWSインフラの構築方法についておはなしします

PowerShellでDSL

登壇者 詳細
杉浦彰(@urasandesu) PowerShellでDSLを設計してみます。
「PowerShellインアクション」にある「PowerShell言語の拡張」を基に、自作ライブラリを例にお話を。

C# からPowerShellを実行してみよう

登壇者 詳細
@tanaka_733 PowerShellはWindowsのCUIツールとして非常に強力ですが、時にはGUIツールから操作をしたいときがあります。
そんなとき、既存のPowerShell資産を活用するためにも、C# コードからPowerShellを操作するお話をします。
単純なコマンド実行にとどまらず、PipeLine実行、スクリプトの実行、リモートコンピューターへの同期・非同期実行について紹介します。

PowerShell Desired State Configuration (DSC) について

登壇者 詳細
牟田口大介(@mutaguchi) Windows Server 2012 R2 とPowerShell 4.0 の新機能である DSC について紹介します。
