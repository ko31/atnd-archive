ヒカ☆ラボ一覧は こちら！！

★講演内容

開発をしているとぶつかる壁-守られない規約や成功しないビルド、起動しないアプリ-や、
ヒットすればするほど課題となる安定したインフラ基盤構築に関して、特に試行錯誤がみられるスマホゲーム界隈。

そんなスマホゲーム開発最前線をはしる、アカツキ社のエンジニア3名にご登壇頂き、
失敗例や成功例を1時間半たっぷりご紹介頂きます！

①RailsとAWSを使って超えた1分間で10万リクエスト処理するインフラ基盤構築の壁
　失敗談も含めご紹介します！　

＜講演内容＞
スマートフォンゲーム市場は家庭用ゲーム市場を超え、まだ上昇トレンドにあります。
その中でヒットするゲームを運用するためには、大量のリクエストを高速に処理できるインフラ基盤が必要です。
アカツキがどのようにして Ruby on Rails と AWS を使って1分10万リクエストの壁を超えたか、
また、その中で失敗したこととは何だったのか、紹介したいと思います。

＜登壇者プロフィール＞
株式会社アカツキ
CTO ハッカーLv.6
田中　勇輔氏(@csouls)

Rubyとインフラが好物で、アプリ開発と並行してアカツキのインフラ基盤を整備してきました。 将来はWeb技術を用いて人類社会の富を大きく増加させるサービスを作り出すことが夢です。 現在は、アカツキのCTOとして、エンジニアが気持よく働けるための環境づくりにも力を入れています。

②知っていると苦労しない、ネイティブアプリ開発でコケるポイント対策

＜講演内容＞
重複コード・守られない規約・成功しないビルド・起動しないアプリ・・・
ネイティブアプリの開発で誰もが一度は経験する辛さをシステマチックに軽減するための
ノウハウや取り組みを紹介したいと思います。

＜登壇者プロフィール＞
宮島　亮氏(@sergeant-wizard）

DvorakとTiling Window Managerが好きです。
最近Hubotで遊んでます。
Dvorak配列に変えたら家族が増え、良い転職もできました。

③マルチバトルを支えるNode.jsの非同期処理手法

＜講演内容＞
サーバークライアント双方向リクエストを捌く際に相性の良いNon-blocking/イベント駆動型の非同期処理。
Nodeが最も得意とするところではありますが、その際必ず取り沙汰されるのがcallback-hellです。
今回は地獄から解放され、気持よく非同期を書くためのモダンな手法を紹介します。

＜登壇者プロフィール＞
谷口　大樹氏（@kidach1）

NodeとRubyが好きです。最近はTypescriptに興味あり。
QwertyからDvorak配列に変えてから腱鞘炎が治り、身長も伸び、彼女もできました。
