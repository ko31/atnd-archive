Test Driven Development for Embedded C 読書会の時間内で写経するのが難しそうなので、写経だけをやる会を開催します。

最近、どーもペアプロするイベントが流行ってるそうなので、ペアプロで写経にします。

■やること
・第8章「Spying on the Production Code」をやります。
・本に書いているテストリストを順番にペアプロでもくもくと写経していきます。
・分かんないことあればその場で解決します。

■持ち物
・Test Driven Development for Embedded C
・開発環境の入ったPC

■参加にあたっての注意事項
・必ずTwitterアカウントか連絡の取れるメールアドレスを設定してください。
・参加費はだいたい1500円弱です（会場代、飲み物代含む）。
・初参加の方でも大丈夫なようにサポートします。ただし、Test Driven Development for Embedded C 読書会に参加したことない方は、第3章と第4章をパラ見して雰囲気だけつかんでおいてください。

■開発環境のインストールについて
・gcc と make は必須です。
・サンプルコードについては大西のブログを参照してください。

■当日の進行
1. 講義
・題材の説明 Light Scheduler
・第8章で使われているTest Doubleの説明（Test Spy、Fake Object）
・クラス図からC言語へのマッピング（クラス、インタフェース、インタフェースの実装）
・link time substitution（リンク時置換）
2. ペアプロでTDD
・今回は全体で進捗を合わせるため、節ごとに一緒に進めていきます。
