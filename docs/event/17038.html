「第1回分散処理ワークショップ in Tokyo」を開催します

■概要

分散処理技術（NoSQL・分散FS・並列プログラミングモデル・大規模データ処理など…）に関するトピックを毎回取り上げ、トピックに沿ったスピーカーをお呼びしてそれについての議論を座談会形式で行う勉強会です。机を円に並べて、参加者も含めて議論できるような場になると良いと思っています。主催者の @doryokujin と @okuyamaoo さんは毎回スピーカーとして参加します。

日程： 2011/07/08（金）18:00 〜 21:00
※勉強会の後は会場での懇親会（有料:21:00 〜 22:30）を予定しております。是非ともそちらもご参加下さい。

■当日の流れ

集合
17:40、17:50に大森ベルポート1Fの北側入口（喫煙所前）にスタッフが迎えに行きますのでそこお待ち下さい。勝手に会場には入れませんのでご注意下さい。遅れて来られる方はツイッターでハッシュタグ #dist_study 、または @nifty_engineer 宛に返信をください。 上記集合場所まで迎えにいきます。Twitterの手段をお持ちで無い方は @doryokujin 宛のメール、 mr.stoicman[at]gmail.com まで連絡を下さい。

ワークショップ
ワークショップ前半の90分は各NoSQLの概要の発表(各15分×5本)、後半は各自テーマや悩みや質問を出し合って全員で自由にディスカッションします。机は円テーブルにする予定です。肩肘のはらない、全員参加の有意義な場になることを期待しています。

懇親会
懇親会は会場内にオードブルとドリンクを持ち込む形になります。是非ともこちらにもご参加下さい、当日の申し込みも可能です。

■目的

分散処理に関するトピックについて、
・運用に関するハマリ所・問題点・Tips等の共有、
・技術に関する詳細な仕組み・特徴・長所/短所の共有
を行う事を目的としています。初心者向けの勉強会ではなく、実運用されている方々同士で情報を共有する場にしたいと思っています。

■今回のトピック

第1回のトピックは、お互いの得意分野であるNoSQLについて、特にスケーラビリティに関する部分にフォーカスして議論したいと考えています。Key-Value・列指向・ドキュメント（・グラフ）それぞれのスケーラビリティ周辺の仕組みや問題点を比較しながら議論して行ければ面白いと思っています。

座談会ではスピーカーがお題をいくつか用意する形で議論を進めていきます。お題に関しては参加者の方からも広く募っていますのでTwitterなどで気軽に声をかけてください。

スピーカー・モデレータ・スタッフ募集中です

NoSQLに詳しく、スピーカーとして参加される方を募集しています。種類は問いません。 @doryokujin までメンションお待ちしております。また、モデレータ・会場スタッフも併せて募集しています。スタッフの皆さんの協力無しには勉強会は成り立つことができません。どうか宜しくお願いします。

■きっかけ

主催者の@okuyamaooさんは分散Key-Valueストア okuyama の開発者で、分散処理系に関しては非常に多くの知識と経験を有しているスゴイ人です。僕は MongoDB JP の代表をやらせてもらっており、実際にデータ解析基盤としてMongoDBを活用しているということもあり、@okuyamaooさんが東京に出張に来られる度にお会いしてokuyamaやmongodbの仕組みや運用上の問題点について、大規模データ処理や次世代リコメンドの手法やあるべき姿などについての話題で盛り上がっていました。そしてこういった実体験や深い知識を持つ方との情報交換の場は非常に有用であることを痛感しており、「それなら勉強会（座談会）を開いて、より多くの方と共有していこう」という運びになり、今回の勉強会発足に至りました。
