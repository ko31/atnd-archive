description

１０回を記念し今回は１日の勉強会です

要件定義を１日かけて演習形式で行います

モデルが使える人はEA（EnterpriseArchitect）を使いUMLで要件定義を行い、普段モデルを書いていない人は「要件のツボ」を使って要件を定義します。

開発してみたいシステムを具体的に要件として落とし込みます。
システム化したいアイディアをお持ちの方はこの機会にアイディアを「見える化」しましょう

午前中はレクチャーとツールを使った基本的な定義方法を説明します。
午後からは各自自分のテーマで要件定義を行います。
途中発表タイムをとり、定義した内容を発表することで定義の精度を発表者が確認します。
うまく説明でき聞いている方にも分かりやすければうまく定義できている事になります。

ツールを使った演習が中心ですので、各自パソコン持参でお願いいたします。
タップをお持ちの方はタップの持参もお願いいたします。

EAで要件定義を行う方はEAとRDRA用テンプレート(http://k-method.jp/temp/dl_template_v1_2.eap)のインストールされているパソコンを持参してください。
要件のツボを使う方は事前にダウンロード（https://vsa.sakura.ne.jp/kaname/5_download/5_download.html）をお願いいたします。

10:00～11:00要件定義（RDRA）の基本的な考え方の説明
11:00～12:30EAを使った定義方法
要件のツボを使った定義方法　説明
13:30～15:00　各自のテーマで演習
15:00～15:30　発表
15:30～16:30　演習続き
16:30～16:50　発表
