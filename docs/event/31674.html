description

近年注目されているNoSQLの中でも、MongoDBは柔軟なクエリや
Replica setによる高い耐障害性、Shardingによるスケーラビリティの
確保などによって、代表的な地位を確保しつつあります。
mongoDB公式サイト はじめに

そのMongoDBを、Rails標準のActiveRecordでRDBを扱うように、
簡単に扱えるようにしたものがMongoidです。ActiveRecordと
同じような扱えるように工夫が凝らされています。
Mongoid公式サイト

しかしドキュメント指向DBであるMongoDBと一般的なRDBでは
以下の点が異なります。

  • サーバサイドでのJOINがない
  • トランザクションがない
  • 埋め込みドキュメントや配列が扱える

このような違いを踏まえた上でCAP定理やACIDなどもおさらいしつつ、
実際Mongoidではどのようにモデルを作るのかという点を掘り下げていこう
と考えております。

MongoDBの勉強会はあってもMongoidの勉強会とか聞いたことなかったので、
正直人が集まるのか非常に不安ですが、NoSQLに興味がある方、使い始めて
いる方、使い倒している方、皆さんと情報交換をしたいと思います。

8/31の金曜の夜、弊社のイベントスペースでお待ちしております。
