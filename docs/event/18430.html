TDDBC Tokyo 1.7 for PHP #tddbc

TDDBC とは、座学だけでなく、ペアプログラミングを通して TDD を実習し、TDD を体感、習得してもらうためのイベントです。
今回は、最近の PHP 界隈での TDD 熱の高まりや、最近開かれた TDDBC でも PHP 希望の方が多くいらっしゃったことから、PHP での言語に限定して TDDBC を開催することになりました。
TDDBC では本来言語を限定せずに行われるものですが、主催の力不足により限定して行わせて頂きます。

今回の TDDBC は PHP限定です。お気を付けください！

目的

以下のような方々を対象に、 お互いに勉強し合うこと を目的とします。

  • テストコード書く意味がいまいち理解できていない
  • TDD に興味はあるが、なかなか実践できない
  • 自動テストを書けるスキルを身につけたい
  • テスティングフレームワークを十分に活用できていない感がある
  • 実際に TDD でソフトウェアを開発したい・見てみたい・体験したい
  • 他のエンジニアはどうやってテスト書いているのか見てみたい・気になる
  • 実際の現場で使える TDD に関するヒントを見つけたい人

(TDDBC Tokyo 1.5 より引用)
