description

システム開発の現場では「ビジネスに貢献する」「ビジネスの価値」「システム化を使い業務を改善する」などビジネスという言葉が当たり前のように出てきます。

でも実際にビジネスをどう表現しますか？

「このシステムはビジネスにどう貢献するのか？」
「会社はどのようなビジネスをしていて今後どうしたいのか？」

このような時に顧客のビジネスを簡潔に表現しするには…

今回はエンジニアがビジネスを表現する方法とシステム化への流れを紹介します。

ビジネスの全体像を「ビジネスモデルキャンバス」で表現し、To-Beシステムに向けてビジネスの改善を１０Types(ビジネスモデルイノベーション)で分析し、組織の責務から仕事への分割を経てビジネスモデルを支えるシステム化へとつなげていきます。

仕事の中で実際にこの手法を使うことは少ないかもしれませんが、この一連の流れを抑えていることで、ビジネスモデルからシステム化への一連の流れをイメージできるようになります。

今回紹介する方式は２社のシステム企画段階でのビジネス整理に実際に活用し、そのフィードバックを受けて体系化したものです。
ユーザ企業でシステム企画を経営者に説明する時にもそのまま活用できます。
