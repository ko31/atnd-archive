description

前回に引き続きシステム化に向けての戦略について考えます。

前回は開発コストを引き下げるためにどのようなアプローチがあるかを整理しながらディスカッションしました。
今回はその続きから行い、時間があれば保守コストを下げるところに取り組みたいと思います。

例えば以下のような要望があるとき…

■情報システム部門の部長から以下の要望があがった

　保守コストを抑えたい
　～

■業務部門から以下の要望があがった

　開発費を安く抑えたい
　システム変更に素早く対応して欲しい
　画面の操作を簡単にして欲しい
　他のシステムと接続性をよくしたい
　～

さあ　あなたがシステム開発の責任者だとしたら、具体的にどのようにアプローチしますか？

以下の言い回しはダメです。

　　・私を信じてください。すばらしいシステムを作ります。
　　・このプロダクトを入れれば開発コストを下げて操作性のいいシステムが作れます。

開発生産性、保守性などシステム開発では毎回語られますが、実際にどのような戦略で臨むのかを考えます。

　どこに注目するか？
　どう調整するか？
　ポイントは何か？

　アーキテクチャとして何を考えるのか？
　業務視点でいかにシステムを分けるか？
　システム視点でいかに分けるか？

今回もグループでのディスカッションと全体でのディスカッションで知見を広げます。

興味のある方はご参加ください。
