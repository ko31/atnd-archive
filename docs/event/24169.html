イベント ユーザー

title イケテルRails3.1勉強会
catch RailsGuides から Routing 周りをみてみよう
description

今回のRails3.1勉強会は 「RailsGuides から ルーティング周りをみてみよう」 と題して、
Rails3.1のルーティング機能を RailsGuides を通してハンズオン形式で学んでいきたいと思います。

事前に以下のページに目を通していただけると良いかと思います。

  • Rails Routing from the Outside In ( 和訳 )
    • 和訳は @usagee_jp が翻訳、公開してくれたものです。ありがとうございます!!

今回の勉強会は、コーディングを行います。
後述の 「必要なもの」、「あると良いもの」 をご確認ください。

是非、ご参加ください。

日時

1月21日(土) 13:00〜18:00
(受付は12:30〜)

場所

住所: 福岡市博多区博多駅東1丁目17番1号　福岡県Ruby・コンテンツ産業振興センター 5階

タイムテーブル

時間 内容
12:30 – 受付
13:00 – Ruby開発サーバの紹介
13:30 – ECサイトの画面とURLを設計する
14:30 – HTTPプロトコルのメソッドとRESTのハナシ
15:00 – ハンズオン
* RailsGuidesをやってみる
* 設計を config/routes.rbにて実際に確認
* Railsで実装
17:00 – クロージング、会場片付け
17:30 – 公開スタッフミーティング

必要なもの

  • ノートPC(要無線LAN)
    • Rails3.1.3 を使用できる状態であること
  • 事前にRailsGuidesの「Rails Routing from the Outside In」に目を通しておいていただけるとありがたいです
    • Rails Routing from the Outside In
    • 日本語訳

あると良いもの

  • Github のアカウント
    • https://github.com/
  • また、ノートPCでGit が使える状態であること

対象

Ruby on Rails に触れたことがある方

コメントください

勉強会の時間配分の参考にさせていただきたいので、希望があれば今回のガイドのどこを重点的にやりたいか以下の選択肢の中からコメントしてもらえると助かります。

  1. ルーターの目的(The Purpose of the Rails Router)
  2. リソースルーティング： Rails のデフォルト(Resource Routing: the Rails Default)
  3. リソースフルでないルーティング(Non-Resourceful Routes)
  4. リソースフルルーティングのカスタマイズ(Customizing Resourceful Routes)
  5. ルーティングの検査とテスト(Inspecting and Testing Routes)
