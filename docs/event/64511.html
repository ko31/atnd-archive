続々と支部も立ち上がりつつあるGCPUG。第二回を開催します！

【テーマ】
Google Cloud Platform の新機能、活用事例、GCPUG をもっと盛り上げよう！

今回は、Google 米国本社からストレージサービス部門のプロダクトマネージャが来日。Google 提供するストレージサービスご紹介していただきます。さらに、モバイル BaaS の Firebase、ビッグデータ処理技術の Cloud Dataflow といった今話題のテクノロジを詳しく解説。GCP の新機能を知る絶好の機会です。

この他にも、GCPの活用事例の紹介、皆さんお楽しみの交流会など盛りだくさんです。GCPUG 第二回に奮ってご参加ください。

◆GCPUG とは
Google Cloud Platform の User Group、略してGCPUGは、GCP を全国に広め、一緒に盛り上げようというグループです。このグループはどなたでも参加出来ます。ギークな方や初心者の方も大歓迎です。GCPUG に参加して GCP の素晴らしさを享受しましょう！

＜こんな方に向いています＞
・GCP って何？美味しいの？という初心者の方。
・GCP はちょっと知ってるけどもう少し詳しく知りたい方。
・GCP を一緒に盛り上げたい方。
・とにかく GCP が大好物な方。

19:00〜19:05 タイトル 『挨拶、GCPUGの活動状況報告』
発表者 吉積情報株式会社 GCP入門本著者　吉積礼敏
概要 第二回を迎えました東京でのGCPUG！　改めてユーザ会の目的等のご紹介と各地での盛り上がり状況と今後の方向性等について、簡単にご紹介します。
19:05〜19:50 タイトル 『新しいストレージサービスについて』
発表者 Google Inc プロダクトマネージャ Cory O’Connor
概要 Google では、BigtableというNoSQLストレージを多くのサービスで利用しています (Gmail, YouTube, Google Analyticsなど) 。オープンソースコミュニティではBigtableの概念を基にHBase、Cassandra、 Accumulo、Hypertableなど、たくさんのプロジェクトが開発されていますが、それと平行にBigtableの開発も進んでいます。OSSとBigtableの違いや、Bigtableの次のステップについて話します。　
19:50〜20:20 タイトル 『FireBaseの概要と使い方』
発表者 Google Inc Ian
概要 モバイルアプリでリアルタイムでデータを同期するのは最高に素敵ですが、実装するのはかなり困難。パフォーマンスや、ネットワークの不安定な場合などいろなことを考慮しないといけません。FirebaseのリアルタイムデータベースAPIを使うと簡単にデータを多数のデバイスと同期ができ、オフラインにも対応できます。Firebaseの概要と使い方について発表します。
20:20〜20:50 タイトル Google Cloud Dataflowが描く「MapReduce後の世界」とは
発表者 Google Inc クラウドプラットフォームチーム デベロッパーアドボケイト 佐藤一憲
概要 Cloud Dataflowは、Googleが過去数年にわたって社内で利用している2つのビッグデータ処理技術、「Flume」と「MilWheel」をGCP上で実装した新しいサービスです。多数の段階からなるデータパイプライン（DAG）を簡潔なコードで記述し、Dataflow環境にデプロイすることで、データの規模に応じて適切な数のGCEインスタンスからなるクラスタを自動で構築、運用できます。また、バッチ処理とストリーム処理の両方に同時に対応。このセッションでは、Cloud Dataflowの概要を解説し、実際の開発環境を用いたデモを紹介します。
20:50〜21:20 タイトル 『GCE最新事情等』
発表者 大橋さん
概要 GCP事例紹介
