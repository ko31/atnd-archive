概要

誰でも参加可能な iOS 、 Objective-C 、 Xcode の読書会形式の勉強会です。

参加条件は特にありません。ご自由にご参加ください！

毎回、独立したテーマがあり、それを一からやりますので、初参加の方も安心です。

あなたもこの機会に iPhone アプリを作れるようになってみませんか？

（クックパッド様のご好意で会場をお借りできましたので、第 7 回も無事開催することが出来ました！ありがとうございます！）

勉強会の内容

  • 各自が事前に宿題資料を読んでくる
  • 会場では資料の内容を振り返り、議論したり疑問点を共有したりする
  • 余った時間は、自由にコード書いたり質問したりする

勉強会の目的

一人で学べる箇所は各自事前に勉強し、一人ではなかなか解決できない箇所を効率的に学んでいくことを目的にしています。

交流や学習自体が目的ではなく、勉強会を通して実践的なコードが書けるようになることが最大の目的です。

この勉強会で効率的に学ぶコツ

事前に、自分が何を理解したいのかを明確にしてその部分を重点的に勉強してきて、この勉強会で議論をすることが効率的に上達するコツです。

宿題の分量は多いし、内容もそれなりに難解だとは思いますので、自分の学びたいこと解決したい悩みの部分だけを重点的に勉強してきて、それを勉強会の中で解決するという方法も大歓迎です。

宿題資料

今回のテーマは、「Blocks とメモリ管理」、「 iOS アプリのテスト手法」です。

（すべて熟読してくる必要はありません。サッと目を通して、疑問点や注意点などを考えておきましょう）

  1. ブロックプログラミングトピック を読む
    • 基本的な Block の使い方、デザインパターン、変数がどのように扱われるかなどが書かれています
  2. メモリ管理プログラミングガイド を読む
    • 資料が若干古く、 ARC については書かれていない。 ARC を使わないメモリ管理に付いての知識が得られます
  3. Transitioning to ARC Release Notes を読む
    • 英語ですが図やソースコードが多く。 ARC について俯瞰して理解することが出来ます
  4. Automatic Reference Counting を読む
    • Xcode で使われているコンパイラ LLVM による ARC の資料です
    • 英語で分量も多いですが、しっかりとした資料です。
  5. Github などで、 iOS アプリのテスト手法について調べてくる

他にも、自分のソースコードを見せたい！というのがあれば大歓迎です！！
