概要

GDC2011で行われた「Automated Testing Roundtable」の議事録内容を参考に、世界の開発環境をスタディ、参加者の環境と比較して議論します。

  • 「Automated Testing Roundtable」の内容紹介
    　http://wiki.igda.org/GDC_2011_-_Automated_Testing_Roundtable
    　を日本語訳したものを紹介
  • ラウンドテーブル形式での議論
    　参加者を適当な人数にグループ分けして行います。
    　議論するトピックの一覧は、「ラウンドテーブルのトピック」をご確認下さい。

    ラウンドテーブルのトピック

    以下の、「Automated Testing Roundtable」のトピックの中から、いくつかを選んで議論します。

    1. テスト

    1-1. シンプルなテスト、スケジューリング、ツール
    ・テストの方法、Tips
    ・CIサーバー、テストスイート管理ツール

    1-2. 入力の扱い

    1-3. レポートと履歴

    1-4. テスト環境

    1-5. アンチパターン、避けるべき事

    1-6. ボット

    1-7. 静的コード解析

    2. インフラ

    2-1. 内製かサードパーティーのソリューションか

    2-2. テレメトリー（遠隔測定）

    2-3. インテグレーション[どこでどうやって、そしてメンテナンス]

    2-4. ブランチとCI（継続的インテグレーション）の戦略；特にブランチの扱いについて（※）

    2-5. 応答（所要）時間

    2-6. 提供方法（集めた情報、テスト結果、など）

    2-7. 分散ビルド

    3. データ収集、マイニング、可視化

    3-1. 普段どのように自動化していますか？
    ・アサーションについて

    3-2. ゲームシステムの上位層のテストについて

    3-3. 投資収益率（Return on Investment）

    3-4. モバイル機器でのテストについて

    3-5. 「充分」vs「やりすぎ」なテスト

    3-6. フレームワークの選択（※）

    3-7. 可視化ツール

    3-8. ユニットテストvs結合テスト、どちらに投資する？（※）

    3-9. webでのテスト

    （※）：元のRTでは議論なし。
