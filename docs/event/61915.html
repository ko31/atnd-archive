今回の趣旨

まともな「表記法の寄せ集め」であれば、おのずとワンセットの認識の枠組みが立ち上がる。（中略）そのあたりの違いを手っ取り早く理解するためには、同一のシステム要件をネタにして同時にモデリングしてみればいい。（渡辺幸三氏ブログ「設計者の発言」2013.01.07「 各種技法の『競演くモデリングライブ』を 」より）

本邦初。異なる開発方法論の実務者によるモデリング発表会です。「T字形ER手法」の鳥谷部氏と「三要素分析法」の渡辺幸三氏に、同一課題にもとづいてその場でモデリングしていただきます。課題の説明後に、業務フローとデータモデルをホワイトボードに描き広げ、その結果を皆で比較検討します。手法毎のモデル様式の違いだけでなく、モデリング過程での「頭の使い方」の違いにも気づいてもらえるでしょう。さらには、手法の違いに関係なく、モデリングすることによる実装合理化の効果までを実感できるでしょう。前代未聞のモデリングライブをお見逃しなく。

花束問題

その場で業務要件を挙げていただきモデリングするというイベントは 第8回 でやりましたが、2時間では一つの手法ですら時間が厳しいという経験を得ました。そのため今回はこの発表会のために練習問題を作りました。

　題して「 花束問題V1.2 」です。もしこの問題の業務モデリングを発表して下さる方が何人かおられるなら再度やりたいと思いますので是非声をかけて下さい。

業務モデリング

この勉強宴会は2010年12月にスタートしました。皆さんのおかげで早くも4年が経過しました。設立の趣旨は「 どんな手法でも良いから業務モデリングしましょう 」という主張です。特に2000年以降にこの業界に入った方はモデリングといえばＵＭＬだと勘違いされている方が多いです。ＵＭＬは「Unified Modeling Language（統一モデリング言語）」ですから書き方しか定義されていません。しかも若い方がＵＭＬで書いているのは「プログラムをどうつくるか」というソフトウエア詳細設計です。

そのあたりは共通フレーム２００７を使って 第21回 でご説明しました。

一時期オブジェクト指向分析という手法が流行りかけましたが行き詰っているように見ています。今回は２つともデータモデリングを主体とした手法ですが、その他の手法の方もぜひこの問題に挑戦して発表して下さい。
