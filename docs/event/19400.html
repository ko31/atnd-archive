＜内容＞

内容が決まりました。

「Gitの準備編」

ゴール：GitHubのアカウントを作って、pushできる準備まで行う。

「基本操作編」

ゴール：自己紹介を入れたREADMEファイルをgithub上に公開する。

「応用編」

ゴール：Pull Requestを送れるところまで

をやる予定です。

もし、今回の内容がレベルが低すぎるよ！という方、講師のエンジニアさんを多めに呼んでいるので、なるべく、知りたいことを個別で教えてあげられるようにしたいのですが、そういった方に満足いただける内容になるかがわかりません。次回はさらに難しいところをやりますので、今回は補欠の方に譲ることもご検討いただけたらと思います。

========

Webデザイナーのための Git 勉強会を開催します

バージョン管理システム Git が使えるデザイナーをもっと増やしたい！と思うエンジニアさんたちが集まって、丁寧に Git が使えるまでデザイナーに教えるというイベントを開催します。

新しいサービスを立ち上げる際、デザイナーがバージョン管理システムを使えることで一気に効率がよくなります。つまり、これからのデザイナーには必須なスキル！

バージョン管理？って方も心配いりません。新しいスキルを身につけたいデザイナー、エンジニアの知り合いが欲しいデザイナー、エンジニアたちに自分のデザインスキルをアピールしたデザイナーは是非、ご参加ください。

参加条件

申し訳ないのですが、ちょっと厳しい参加条件があります。

Webデザイナーである（デザインが出来る、コーディングが出来る）
Macを持っていて、Macを持ってこれる（windowsでもVMwareでubuntuを動かせればOK）

以上の条件を満たしている方限定になります。

自分のMac（アダプタを忘れずに）
