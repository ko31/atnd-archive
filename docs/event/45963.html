description

OpenStreetMap( http://www.openstreetmap.org/ ) という、
Wikipediaのようにみんなで自由に使える地図データを作ろう！という英国発のプロジェクトがあります。
世界中の人々が自分の知っている情報を地図に投入することで、皆で世界地図を作ろう！という壮大なプロジェクトです。
欧州を中心に、Google Mapsと同等かそれ以上に詳しい地域もあります。

1週間前にオープンしたばかりのお店も、解体されてしまったビルも、数分の間に地図に反映させることができます。
全世界で100万人を超える地図編集者達が、日々地図を描いています。

前回金沢で行ったもくもく勉強会にて、HelloWorld的なJOSMプラグインまで出来たので、これをベースに便利なプラグインを作ります。

一例ですが、以下のテーマから選んでもくもくと作業しましょう。
【開発者向け】
・郵便番号から住所を取得して埋めるJOSMプラグインの開発(Java)
・name入力時にname:ja_rm,name:ja_kanaを埋めるJOSMプラグインの開発(Java)
・OpenStreetMapのオフラインレンダリングをしてみよう(Flash, python, .NETなど)
・APIを使ってみよう(JavaScript, Flash, Java, Objective-C, ..etc プログラム言語問わず)

【デザイナー向け？】
・地物アイコンを作ろう(もしくは Public Domainなアイコンを探そう)。
OSMは欧州発のプロジェクトのため、基本的には日本にしかないような地物のアイコンがありません。
OpenStreetMap Japanでは現在日本独自のレンダリングを進めており、アイコンを募集中です。
あなたのアイコンがOSM Japanの地図に採用されるかも！？

【地図を描きたい方向け】
・基盤地図情報を使って、ひたすら地図を描こう。

自由にOSMと戯れましょう。

もくもくについて

ひたすらOSMの地図を描くも良し、アプリを開発するも良し、発表するも良しです。
自分のペースでOSMと戯れましょう。

僕は以下のようなことに興味があるので、例えばこんなことやりたいです。

  • タイルサーバから地図タイルを取得するのではなく、アプリ内でOSMファイルから地図レンダリングしてみる。
    →標準では地名は現地語でレンダリングされるが、クライアントでレンダリングすることで簡単にロケールを切り替えられるように。
    →支店(branch)タグがレンダリングされないため、東横インXX駅東口1号館と東横インXX駅東口2号館が一見して区別できない。branchのレンダリングも切り替えたい。
  • JOSMプラグイン作り
  • 地図サーバを立て、閉じたイントラ環境でもWeb地図を使えるようにしてみる。

その他にも要望があればこんな話なら出来ます。

  • オープンソースの帳票設計ツールiReportを使い、OpenStreetMapの地図画像を緯度・経度を指定してPDFに埋め込んでみよう。
