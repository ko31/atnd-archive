description
「ビジネスルール」第２段は実際に手を動かして要件定義のモデリングを行います。
ビジネスキャンバスを題材にそのビジネスモデルを支援するシステムの要件定義を行います。

ポイントはビジネスキャンバスからシステムのイメージをつかむこと

３時間でシステムの全体像がつかめる要件定義を行います。
最初にフォーカスする部分を絞り、手処理の部分とシステム化する部分を切り分けます。
短時間でモデリングするためにはビジネスに必要な基本機能のパターン（ビジネステンプレート）をビジネスキャンバスに当てはめ、手処理とシステム化の識別を行います。

ビジネスキャンバスには出発点となる情報があり、それをビジネステンプレートに当てはめることで必要な機能が見えてきます。それを出発点にRDRAのモデルでシステムに必要な機能を洗い出していきます。

※ビジネステンプレート（仮称）は私がコンサルの現場で使ってきたものを整理したもので、ビジネスに必要となる機能をポンチ絵で示したものです。業務知識がなくても常識の範囲でビジネス機能を理解できるようにまとめたものです。

３時間という短時間に粒度を揃えビジネスキャンバスでうたわれている価値を実現するシステムを定義することを目指します。

題材:「ビジネスモデルジェネレーション」72ページ「レゴファクトリー」

この１枚のビジネスキャンバスを頼りにイメージをふくらませ、システムのイメージを作りあげます。

・モデリングが初めての方はモデリングの醍醐味を味わうことができます。
・ビジネスキャンバスに興味のあるかたはシステム化の様子が分かります。

当日は私の方でファシリテーションしながらモデリングを行うので、モデリングに不慣れな方もお気軽にご参加ください。もちろん自分でモデリングしたい方は各自のパソコンでモデリング可能です

参加者の方はビジネスキャンバスからシステム化のアイディアを出していただき、モデルとして形にしていきます。

出たとこ勝負のモデリングです　３時間にコミットしみんなでゴールを目指しましょう！</3>
