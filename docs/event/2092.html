スピードは正義です。

いまやネットブックですらマルチコアであり、近い将来にはあなたの iPhone もマルチコアになることでしょう。デスクトップにおいては GPU という超並列計算機が既にコモディティと化していますし、サーバサイドについてはいわずもがな。

そんなマルチコア時代でプログラマとしてこの先生きのこるためには、並列プログラミングを身につける必要があります。並列プログラムを書けないと、速いソフトウェアがつくれない。速いプログラムが書けないと、正義を成すことができない。これでは困ります。

そういうわけで、並列プログラミングはいまや、モバイルからサーバサイドに至るまでのすべての開発者の必須科目になってしまったのでした。やれやれ。しかし、並列プログラミングはなかなかにむずかしく、敬遠されがちです。いやはや。

そういうわけで、正義を成すために並列プログラミングについて語りませんか。

内容は、基礎からマニアックなところまで。アカデミックなだけでなく、プラクティカルな並列話を聞いてみたいです。たとえば、ぼくはサーバ側の並列事情とかまったく知らないので聞いてみたい。話したい方は意思表明をコメントなどしてください。聞きたいトピックスについても、どしどしどうぞ。
