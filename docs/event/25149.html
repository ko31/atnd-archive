Scalaとオブジェクト指向とデザインパターンと関数型プログラミングを学んでいます。

お題は 結城浩 著『増補改訂版Java言語で学ぶデザインパターン入門』 から。

今回は、下記を対象とします。

  • Flyweight ― 同じものを共有して無駄をなくす
  • Proxy ― 必要になってから作る
  • Command ― 命令をクラスにする

進め方としては大体以下の通りです

  • 勉強会当日までに
    • 『＊１・・・このデザインパターンはこういう動機・こういうケースの時に書くとこういうメリットがあるよ』ということを優しい人が 名古屋Scalaの練習場掲示板 に書いてくれることを期待。
    • 該当デザインパターンのJavaのサンプルコードをScalaっぽく（ある種写経気味に）書く。
    • ＋αで『＊１』を踏まえた上で、そもそもこういう風に書くとScalaっぽいよ！というのを書く。
    • その他、該当デザパタに関して「これって実際に使ったことある？」とかの疑問なども書く。
  • 勉強会当日
    • 自己紹介。
    • デザパタに取り掛かる前に『＊１』について参加者で認識合わせをする。
    • 名古屋Scalaの練習場掲示板 に書かれたコードや意見を題材にディスカッション。
    • （この文法ってどういう意味？こっちの書き方の方が意図が伝わりやすいのでは？などなど）
    • 該当パターンを下記の３つに分類してみる。
  • Scalaでも重要である
  • Scalaだとあまり使わないが、なお重要な場合はありえる
  • Scalaだと必要ない
  • 勉強会終了後
    • 行ける人は懇親会！

疑問だとか、「こんなコードどうよ！」とかは
名古屋Scalaの練習場掲示板 にご記載下さい。
